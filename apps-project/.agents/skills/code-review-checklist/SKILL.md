# コードレビュー チェックリスト

## 🔴 Critical（必ず確認）

### セキュリティ
- [ ] SQLインジェクション・XSS・CSRF の脆弱性がないか
- [ ] 認証・認可が適切か（未ログインユーザーがアクセスできるエンドポイントがないか）
- [ ] シークレット・パスワードがハードコードされていないか
- [ ] 入力値のバリデーションがあるか

### 正確性
- [ ] 仕様通りに動くか
- [ ] エッジケース（空・null・0・境界値）が考慮されているか
- [ ] エラーハンドリングが適切か（握りつぶしていないか）
- [ ] 非同期処理のエラーが捕捉されているか

### データ
- [ ] DBのN+1問題がないか
- [ ] トランザクションが必要な箇所で使われているか
- [ ] データの整合性が保たれるか

---

## 🟡 Warning（直すべき）

### 保守性
- [ ] 関数・変数名が意図を表しているか
- [ ] 関数が長すぎないか（目安：30行以上は要注意）
- [ ] 重複したコードがないか
- [ ] マジックナンバー・マジック文字列が定数化されているか

### テスト
- [ ] 重要なロジックにテストがあるか
- [ ] テストが実装の詳細ではなく振る舞いをテストしているか

### 型・インターフェース
- [ ] 型が適切に定義されているか（any を使っていないか）
- [ ] 公開APIの型が明確か

---

## 🔵 Suggestion（あれば良い）

### パフォーマンス
- [ ] 不要な再計算・再レンダリングがないか
- [ ] キャッシュで改善できる箇所がないか

### 可読性
- [ ] コメントが「何をしているか」ではなく「なぜそうしているか」を説明しているか
- [ ] 複雑なロジックに説明コメントがあるか
